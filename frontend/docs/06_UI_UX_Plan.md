# HiddenPro MVP 핵심 UI/UX 화면 설계 지시서 (06_UI_UX_Plan)

## 📌 개발 AI (Antigravity) 필수 숙지 사항

이 문서는 단순한 디자인 가이드가 아니라 **비즈니스 수익 창출을 위한 핵심 화면 명세서**입니다. 모든 프론트엔드 컴포넌트는 아래 명시된 '버튼 위치', '데이터 노출 방식', '유저 행동 유도(CTA)' 규칙을 100% 준수하여 코딩해야 합니다. 예쁘게 만드는 것보다 **결제 전환율(Conversion Rate)**이 최우선입니다.

---

### 📱 1. [고객] 동적 챗봇 요청서 (Dynamic Request Flow)

고객이 서비스 요청을 포기하지 않고 끝까지 완료하게 만드는 것이 핵심입니다.

- **상단 UI:** 현재 진행 중인 카테고리명과 함께 **'프로그레스 바(진행률)'**를 상단에 고정 노출하여 완료가 임박했음을 시각적으로 줍니다.
- **중앙 챗봇 영역:** `04_JSON_Form_Schema.md`를 기반으로 질문 하나당 하나의 화면(또는 말풍선)만 띄웁니다. 스크롤이 긴 설문지 형태는 절대 금지합니다.
- **하단 이탈 방지 로직 (가장 중요):** 모든 질문에 답하기 전까지는 **절대 로그인 창을 띄우지 마십시오.** 마지막 답변이 끝난 직후 **[견적 받을 고수 매칭하기]**라는 강력한 CTA 버튼을 띄우고, 그 버튼을 눌렀을 때만 소셜 로그인/가입 창(BottomSheet)이 올라오도록 설계합니다.

### 💰 2. [고수] 받은 요청 리스트 및 견적 발송 (Pro Quote Bidding)

고수의 결제 조급증(FOMO)을 자극하여 캐시를 소진하게 만드는 플랫폼의 핵심 수익 창출 화면입니다.

- **리스트 화면 (FOMO 자극):** 각 고객의 요청서 카드 우측 상단에 **현재 입찰 중인 고수 수 (예: 💡 2명 / 5명 견적 발송 완료)**를 눈에 띄게 배치하여 경쟁 심리를 자극하십시오. (48시간 남은 타이머도 빨간색 텍스트로 표기)
- **상세 열람 화면 (JSON 파싱):** 고객이 작성한 JSON 응답 데이터를 파싱하여 읽기 편한 리스트 형태로 보여줍니다.
- **견적 발송 버튼 (과금 트리거):** 화면 우측 하단에 **[OOO 캐시로 견적 보내기]** 버튼을 **플로팅(Floating) 형태로 화면에 항상 고정**시킵니다. 고수가 스크롤을 내려도 이 버튼은 절대 사라지면 안 됩니다. 버튼 클릭 시 `Cash_Ledger` 차감 API가 즉시 호출되어야 합니다.
- **고수 전문성 보장 및 어뷰징 방지 정책 (Service Limit Rule):**
  - 1명의 고수(Pro)가 프로필에 등록할 수 있는 '제공 서비스(3뎁스 상세 카테고리)'는 **최대 5개**로 엄격히 제한합니다.
  - 이는 무분별한 문어발식 서비스 등록(심부름센터화)을 방지하고, 특정 분야의 진짜 전문가를 고객과 매칭하여 플랫폼의 전환율과 신뢰도를 높이기 위한 필수 제약 사항입니다.

### 💬 3. [공통] 실시간 채팅 및 리뷰 (Chat & Review)

플랫폼 외부로의 우회 거래를 막고, 거래 성사를 증명하는 공간입니다.

- **채팅방 상단:** 상대방의 프로필 정보와 함께, 고수가 보냈던 '견적 금액'을 상단에 고정 패널로 띄워둡니다.
- **매칭 확정 스위치 (1방향 리뷰 트리거):** 채팅방 상단 중앙에 **[매칭 확정하기]** 버튼(고수/고객 모두 클릭 가능)을 둡니다. 이 버튼이 눌려 상태가 'MATCHED'로 변환된 방에서만, 고객 화면 하단에 **[이 고수에게 리뷰 남기기]** 버튼이 생성되어야 합니다.
- **빈 방 유도 로직:** 채팅 탭에 진입했는데 생성된 방이 하나도 없을 경우, 빈 화면 대신 **고객은 [새로운 견적 요청하기], 고수는 [받은 요청 보러가기] 버튼**을 큼직하게 노출시켜 액션을 유도합니다.
