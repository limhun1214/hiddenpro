# MCP 도구 활용 원칙 (Vibe Coding Toolchain)

프로젝트 개발 중 다음 상황이 발생하면, 에이전트는 사용자의 개입을 요구하지 말고 즉각적으로 해당하는 MCP 도구를 활용하여 문제를 해결해야 한다.

## DB 스키마 확인 및 쿼리 (`supabase-postgres`)

- 코드 작성 전 docs 문서와 함께 실제 테이블/컬럼/타입을 직접 조회하여 교차 검증하라.
- 읽기 전용으로 동작할 경우, 스키마 변경(DDL)이나 데이터 조작(DML)이 필요할 때 **절대 사용자(디렉터)에게 대시보드 수동 조작을 요구하지 마라.** 대신 마이그레이션 SQL 스크립트를 작성하고 Supabase CLI나 Node.js 스크립트를 통해 자동으로 실행되도록 파이프라인을 구축하라.

## 인프라 제어 및 엣지 배포 (`cloudflare`)

- 도메인 설정, Workers/Pages 배포 상태 확인, D1(DB)/KV/R2 리소스 조회 및 관리가 필요할 때 적극 활용하라.
- Cloudflare 전면 사용 승인됨 (2026-03-27 디렉터 공식 승인)

## 형상 관리 (`git`)

- 현재 브랜치 상태 파악, 커밋 이력 조회, 변경 사항(Diff) 확인 및 코드 푸시가 필요할 때 활용하라.

## 프로젝트 장기 기억 (`memory`)

- 핵심 아키텍처 결정 사항, 데이터 모델링 구조, 비즈니스 규칙 등 세션을 넘어서 유지해야 할 중요한 컨텍스트가 발생하면 지식 그래프에 즉시 기록하고 조회하라.

## UI/UX 디자인 (`stitch`)

- 프론트엔드 컴포넌트 설계, 디자인 시스템 기반 화면 생성 및 UI 수정이 필요할 때 활용하여 일관성 있는 화면을 구축하라.
